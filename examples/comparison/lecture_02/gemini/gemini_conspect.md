## Введение в алгебру логики

Формальная логика основывается на принципах, заложенных еще в Древней Греции. Согласно этой традиции, любое атомарное высказывание имеет строго одно из двух состояний: оно либо истинно, либо ложно. Состояний, являющихся одновременно истинными и ложными (или не истинными и не ложными), в данной системе не существует.

Оценка истинности высказывания всегда зависит от выбранной понятийной или аксиоматической системы (языка). Алгебра логики абстрагируется от внутреннего содержания (смысла) атомарных высказываний и работает с составными высказываниями, оперируя исключительно их логическими значениями: истиной ($1$) и ложью ($0$).

## Логические функции

Любая логическая функция представляет собой отображение множества допустимых значений переменных на множество значений функции. Поскольку логическая переменная может принимать только два значения ($0$ или $1$), область определения логических функций конечна. Это позволяет задавать их таблично (с помощью таблиц истинности).

Для функции одной логической переменной $f(x)$ область определения состоит из двух точек ($0$ и $1$). Всего существует $4$ варианта таких функций:
*   Тождественный ноль (противоречие).
*   Тождественная единица (тавтология).
*   Тождественная функция: $f(x) = x$.
*   Отрицание: $f(x) = \bar{x}$.

Для логической функции двух переменных $f(x, y)$ область определения содержит $2^2 = 4$ строки. Количество всевозможных функций от двух переменных вычисляется по правилу умножения и составляет $2^4 = 16$. В общем случае количество логических функций от $n$ переменных составляет $2^{2^n}$ (например, для трех переменных это $2^8 = 256$ функций).

## Базовые логические операции

**Определение:** Отрицание (инверсия) — унарная операция, которая меняет значение логического высказывания на противоположное. Обозначается чертой над переменной: $\bar{A}$.

**Определение:** Конъюнкция (логическое умножение, операция И) — бинарная операция, которая истинна тогда и только тогда, когда истинны оба операнда. Обозначается как $A \cdot B$.

**Определение:** Дизъюнкция (логическое сложение, операция ИЛИ) — бинарная операция, которая истинна, если истинен хотя бы один из операндов. Обозначается как $A + B$. При сложении двух истин результат остается истиной ($1 + 1 = 1$), так как логическое значение не может превышать единицу.

**Определение:** Исключающее ИЛИ (XOR, сложение по модулю 2, антиэквиваленция) — бинарная операция, истинная только в том случае, если значения операндов не совпадают. Обозначается как $A \oplus B$. В отличие от классической дизъюнкции, для этой операции $1 \oplus 1 = 0$.

**Определение:** Импликация (следование) — операция, связывающая посылку (левую часть) и вывод (правую часть). Обозначается как $A \to B$. Импликация ложна только в одном случае: когда из истинной посылки следует ложный вывод. Из ложной посылки логически может следовать как истинный, так и ложный вывод, при этом само рассуждение остается математически истинным.

**Определение:** Эквиваленция — операция проверки на равенство. Обозначается как $A \equiv B$ или $A = B$. Выражение истинно, если операнды одновременно ложны или одновременно истинны.

## Законы алгебры логики и нормальные формы

Любое сколь угодно сложное логическое высказывание можно записать, используя лишь три базовые операции: конъюнкцию, дизъюнкцию и инверсию. Эти три операции образуют полную систему. 

**Определение:** Дизъюнктивная нормальная форма (ДНФ) — это способ записи логической функции в виде логической суммы произведений, где каждое слагаемое отвечает за точное совпадение с одной строкой таблицы истинности, в которой функция принимает значение $1$.

**Пример:** Если функция трех переменных $f(x, y, z)$ должна выдавать $1$ на наборе переменных, где $x=0, y=0, z=0$, то соответствующее слагаемое записывается через отрицание каждой переменной: $\bar{x} \cdot \bar{y} \cdot \bar{z}$. Если функция равна $1$ при $x=0, y=1, z=0$, слагаемое примет вид $\bar{x} \cdot y \cdot \bar{z}$. Итоговая функция получается путем сложения (дизъюнкции) всех таких слагаемых.

Для упрощения логических выражений используются законы алгебры логики:

*   Взаимодействие с константами:
    *   $0 + x = x$
    *   $1 + x = 1$
    *   $0 \cdot x = 0$
    *   $1 \cdot x = x$
*   Законы повторения:
    *   $A \cdot A = A$
    *   $A + A = A$
*   Закон третьего лишнего:
    *   $A + \bar{A} = 1$
    *   $A \cdot \bar{A} = 0$
*   Закон поглощения:
    *   $A \cdot (A + B) = A$
    *   $A + A \cdot B = A$
*   Ассоциативность (скобки можно опускать):
    *   $A \cdot (B \cdot C) = (A \cdot B) \cdot C$
    *   $A + (B + C) = (A + B) + C$
*   Коммутативность:
    *   $A \cdot B = B \cdot A$
    *   $A + B = B + A$
*   Дистрибутивность (раскрытие скобок работает как для умножения, так и для сложения):
    *   $A \cdot (B + C) = A \cdot B + A \cdot C$
    *   $A + B \cdot C = (A + B) \cdot (A + C)$
*   Законы де Моргана (отрицание суммы и отрицание произведения):
    *   $\overline{A + B} = \bar{A} \cdot \bar{B}$
    *   $\overline{A \cdot B} = \bar{A} + \bar{B}$
*   Закон двойного отрицания:
    *   $\overline{\overline{A}} = A$
*   Раскрытие импликации:
    *   $A \to B = \bar{A} + B$

## Логические значения и ветвления в Python

В языке программирования Python для логических значений используется тип `bool` с двумя константами, которые пишутся с большой буквы: `True` (истина) и `False` (ложь).

Для объединения логических высказываний применяются операторы `and`, `or` и `not`. Логические операторы имеют более низкий приоритет, чем операторы сравнения (`==`, `<`, `>`, `%` и др.), поэтому базовые математические сравнения можно писать без дополнительных скобок.

Оценка логических выражений внутри цикла позволяет аккумулировать результаты и заменять громоздкие условные конструкции `if`.
**Пример:** Поиск хотя бы одного числа, делящегося на 10. Если переменная `flag` изначально равна `False`, в цикле можно использовать логическое сложение:
```python
flag = flag or (x % 10 == 0)
```
**Пример:** Проверка того, что все числа делятся на 10. Изначально `flag` устанавливается в `True`, а в цикле используется логическое умножение:
```python
flag = flag and (x % 10 == 0)
```

Для управления потоком выполнения используются условные конструкции ветвления.
**Пример:** Каскадное ветвление позволяет избежать глубокой вложенности блоков кода, используя оператор `elif`. Если требуется разделить числовую прямую на диапазоны:
```python
if x < 0:
    print("A")
elif x < 5:
    print("B")
elif x < 10:
    print("C")
else:
    print("D")
```
В конструкции с `elif` каждое последующее условие подразумевает, что предыдущие оказались ложными. В блоке `elif x < 5:` математически гарантировано, что $x \ge 0$.

**Пример:** Вложенное ветвление, напоминающее двоичное дерево решений. Используется для разделения логики на подслучаи, например, при определении координатной четверти точки (при условии, что $x \neq 0$ и $y \neq 0$):
```python
if y > 0:
    if x > 0:
        print("I")
    else:
        print("II")
else:
    if x < 0:
        print("III")
    else:
        print("IV")
```
Обе конструкции эффективны, однако каскадное ветвление предпочтительнее при проверке цепочки взаимоисключающих условий, так как линейный код визуально легче анализировать.
