# Языки программирования и их применение в криптографии

**Концепция:** Математическая основа криптографии — ключевой этап перед изучением любых криптографических протоколов. Без понимания математического аппарата (теории чисел, теории групп, эллиптических кривых) изучение алгоритмов RSA, Эль-Гамаля или эллиптической криптографии теряет смысл и становится бессистемным.  

**Алгоритм:**  
1. Начинаем с **RSA** — изучаем теорию сравнений по модулю, функцию Эйлера φ(n), выбор открытых и закрытых ключей на основе разложения n = p·q.  
2. Переходим к **Эль-Гамаля** — вводим понятие мультипликативной группы Z_p^*, порядок элемента, дискретное логарифмирование как основу безопасности.  
3. Затем — **эллиптическая криптография**: определяем эллиптическую кривую над конечным полем, операцию сложения точек, порядок подгруппы, изоморфизм между Z_p и E(F_p).  

**Плюсы:**  
- Чёткая структура: математика → протокол → реализация.  
- Минимизация ошибок при изучении — каждый шаг логически обоснован.  
- Возможность повторного обращения к математическим основам при переходе к новым темам (например, квантовой криптографии).  

**Минусы:**  
- Ограниченное время: не успеваем полностью раскрыть эллиптическую криптографию и квантовую часть из-за объёма материала.  
- Отсутствие практических примеров на доске — только теоретическая проработка.  
- Риск потери мотивации у студентов при отсутствии немедленного применения математики к протоколам.  

**Концепция:** Выбор инструментов работы — принципиально важен для эффективности. Мы используем **Far Manager** как основной файловый менеджер, поскольку он позволяет выполнять все операции (копирование, переименование, создание папок) исключительно с клавиатуры, без мыши. Это ускоряет работу в 2–3 раза по сравнению с проводником Windows или Total Commander.  

**Алгоритм:**  
1. Все действия — через горячие клавиши: `F4` — открыть папку, `Ctrl+D` — создать директорию, `F5` — обновить список.  
2. Работа без мыши — стандартная практика для системных администраторов и разработчиков.  
3. При необходимости — кратковременное использование мыши (например, при работе с графическим интерфейсом), но основное внимание — клавиатурным комбинациям.  

**Плюсы:**  
- Высокая скорость навигации по файловой системе.  
- Полный контроль над процессом без визуальных задержек.  
- Подходит для автоматизации рутинных задач (например, подготовка директорий под проекты).  

**Минусы:**  
- Кривая обучения: новичкам требуется время на запоминание клавиш.  
- Отсутствие графической подсказки — ошибки возможны при первом использовании.  
- Не поддерживается в Linux по умолчанию — требуется установка через репозиторий или сторонние сборки.  

**Концепция:** Криптография — не изолированная дисциплина, а часть более широкой IT-культуры. Мы не будем углубляться в квантовую криптографию (QKD), поскольку она требует принципиально иной парадигмы: вместо вычислений на классических машинах — физические процессы с запутанными фотонами и детекторами.  

**Алгоритм:**  
1. Классическая криптография (RSA, Эль-Гамаль) — основана на вычислительной сложности задач (факторизация, дискретный логарифм).  
2. Квантовая криптография — основана на **законах квантовой механики**: невозможность клонирования состояния, коллапс при измерении.  
3. Протокол BB84 — отправка кодов в случайных базисах, измерение с совпадением базиса, обнаружение прослушивания по изменению ошибок.  

**Плюсы:**  
- Квантовая криптография обеспечивает **теоретически безусловную безопасность** (при идеальных условиях).  
- Не подвержена атакам на основе вычислительной мощности (в отличие от RSA при росте битности ключей).  

**Минусы:**  
- Требует специализированного оборудования: квантовые источники, детекторы, оптоволокно.  
- Высокая стоимость и низкая масштабируемость.  
- Не заменяет классические методы — используется в ограниченных каналах (например, между банками или правительственными структурами).  

**Концепция:** Учебный процесс строится на **последовательности**: математика → протокол → практика. Мы не будем «прыгать» по темам, даже если часть материала кажется интересной (например, квантовая криптография), — это нарушит логику курса и приведёт к поверхностному пониманию.  

**Алгоритм:**  
1. Сегодня: **математика RSA** — сравнение, модулярная арифметика, выбор ключей.  
2. Завтра: **протокол RSA** — шифрование/расшифровка, подпись, уязвимости (например, атака по времени).  
3. Послезавтра: **Эль-Гамаль** — аналогия с RSA, но на основе дискретного логарифма.  

**Плюсы:**  
- Чёткая логика изложения.  
- Возможность систематической проверки знаний через контрольные вопросы после каждого блока.  
- Подготовка к реальным задачам: защита данных, цифровые подписи, безопасная передача информации.  

**Минусы:**  
- Ограничение по времени — не успеваем охватить все темы в идеальном объёме.  
- Отсутствие лабораторных работ на доске — только теоретическая проработка.

**Концепция:** Классическая криптография строится на трёх основных математических направлениях: теории чисел, дискретном логарифмировании и эллиптических кривых. Каждый алгоритм — RSA, Эль-Гамаль, эллиптическая криптография — требует предварительного изучения соответствующей математической основы перед изложением протокола. Без этого дальнейшее объяснение становится бессмысленным.  

**Алгоритм:**  
1. Изучается теория чисел → затем RSA (асимметричное шифрование на основе факторизации больших чисел).  
2. Изучается дискретное логарифмирование → затем алгоритм Эль-Гамаля (основан на сложности вычисления логарифма в мультипликативной группе).  
3. Изучается теория эллиптических кривых над конечными полями → затем эллиптическая криптография (наиболее эффективная по соотношению безопасности и размера ключа, но требует глубокого математического понимания).  

**Плюсы:**  
- Чёткая логика изложения: математика → протокол.  
- Возможность систематической проверки знаний через контрольные вопросы после каждого блока.  
- Подготовка к реальным задачам: защита данных, цифровые подписи, безопасная передача информации.  

**Минусы:**  
- Ограничение по времени — не успеваем охватить все темы в идеальном объёме (особенно эллиптическая криптография и квантовая).  
- Отсутствие лабораторных работ на доске — только теоретическая проработка.  

**Концепция:** Курс рассчитан на 30 занятий, читается на четвёртом курсе. Объём материала позволяет изучить:  
- RSA (на основе теории чисел),  
- Эль-Гамаль (на основе дискретного логарифмирования),  
- Эллиптическую криптографию (как «писк» — актуальное, но не успеваем).  

**Алгоритм:**  
1. Математическая основа → описание протокола.  
2. Повторяем для каждого направления: теория чисел → RSA; дискретный логарифм → Эль-Гамаль; эллиптические кривые → эллиптическая криптография.  
3. После — переход к квантовой криптографии, где концепция кардинально меняется (например, протокол BB84).  

**Плюсы:**  
- Структурированный подход: математика всегда предшествует протоколу.  
- Акцент на практическое применение в защите данных и цифровых подписях.  

**Минусы:**

**Концепция:** Выбор тем зависит от аудитории: студенты формулируют, что хотят изучать; преподаватель определяет, что **не нужно**. Таким образом, курс адаптируется под интересы группы.  

**Алгоритм:**

**Плюсы:**

- Возможность использовать OpenSSL и другие инструменты без необходимости писать код с нуля.  

**Минусы:**

# Доказательства с нулевым знанием и оригинальность учебных материалов

**Концепция:** Асимметричная криптография — основа современной цифровой безопасности, позволяющая каждому пользователю иметь пару ключей: открытый (публичный) и закрытый (приватный). Открытый ключ доступен всем и используется для шифрования сообщений или проверки подписей; закрытый ключ хранится в секрете и применяется только для расшифровки или создания цифровых подписей.  

**Алгоритм:**  
1. При необходимости отправить зашифрованное сообщение — отправитель использует **открытый ключ получателя**.  
2. Получатель расшифровывает сообщение с помощью **своего закрытого ключа**.  
3. Для цифровой подписи: отправитель подписывает данные своим **закрытым ключом**, а любой может проверить подпись через его **открытый ключ**.  

**Плюсы:**  
- Масштабируемость: не нужно хранить уникальную пару ключей для каждой пары участников (в отличие от симметричной криптографии).  
- Безопасность: потеря открытого ключа не угрожает системе — он и так публичен.  
- Поддержка цифровых подписей без доверия к третьему лицу.  

**Минусы:**  
- Медленнее симметричных алгоритмов из-за вычислительной сложности операций с большими числами (RSA, ECC).  
- Уязвимость перед квантовыми атаками: при появлении мощных квантовых компьютеров асимметричные схемы (включая RSA и эллиптические кривые) могут быть взломаны за полиномиальное время.  

**Концепция:** Симметричная криптография — устаревшая по масштабу, но эффективная по скорости модель, где один и тот же ключ используется как для шифрования, так и для расшифровки. Применяется в основном внутри доверенных систем (например, между государствами или устройствами одной сети).  

**Алгоритм:**  
1. Стороны заранее обмениваются секретным ключом (через безопасный канал или протокол типа Diffie-Hellman).  
2. Все сообщения шифруются и расшифровываются с использованием **одного и того же ключа**.  

**Плюсы:**  
- Высокая скорость обработки данных — идеальна для потокового шифрования (например, TLS, IPsec).  
- Простота реализации на аппаратном уровне.  

**Минусы:**  
- Требует надежной передачи ключа между сторонами.  
- При росте числа участников количество необходимых ключей растёт квадратично: N(N−1)/2 пар ключей — неприемлемо при миллиардах пользователей.  

**Концепция:** Хэширование — односторонняя функция, преобразующая произвольные данные в фиксированный по размеру "отпечаток" (хеш). Используется для проверки целостности данных и построения цифровых подписей.  

**Алгоритм:**  
1. Входные данные (файл, сообщение) подаются на вход хэш-функции (например, SHA-256).  
2. Результат — уникальный хеш (например, `a94a8fe5ccb19ba61c4c0873d391e987982fbbd3`).  
3. При изменении даже одного бита входных данных хеш кардинально меняется.  

**Плюсы:**  
- Быстрое вычисление и проверка целостности.  
- Необратимость: по хешу невозможно восстановить исходные данные.  

**Минусы:**  
- Уязвим к коллизиям (разным данным — одинаковый хеш), хотя современные стандарты (SHA-3) минимизируют этот риск.  
- При квантовых атаках возможны атаки типа **Grover’s algorithm**, ускоряющие поиск коллизий.  

**Концепция:** Квантовые угрозы — главная причина перехода от асимметричной криптографии к постквантовым решениям. Современные алгоритмы (RSA, ECC) не выдерживают атаки на квантовых компьютерах с тысячами кубитов.  

**Алгоритм:**  
1. При появлении квантового процессора с достаточным числом стабильных кубитов — взлом RSA и ECC становится возможен за полиномиальное время (алгоритм Шора).  
2. Решение: переход на **постквантовые криптосистемы**, устойчивые к таким атакам (например, на основе решёток, хэш-функций или кодовой криптографии).  

**Плюсы:**  
- Устойчивость к квантовым атакам.  
- Поддержка долгосрочной безопасности (20+ лет).  

**Минусы:**  
- Низкая скорость по сравнению с классическими алгоритмами.  
- Отсутствие широкого внедрения и стандартов (на 2024 год — NIST только завершает отбор постквантовых алгоритмов).  

**Концепция:** Pipeline организации учебного материала — структура курса, где каждая папка (`language`, `programs`, `course`) содержит строго определённые типы контента: теоретические материалы, примеры решений, шаблоны оформления и задачи для самостоятельного выполнения.  

**Алгоритм:**

**Концепция:** Асимметричная криптография — это подход, при котором для шифрования и проверки используется пара ключей: **открытый** (публичный) и **закрытый** (приватный). Открытый ключ может быть известен всем участникам системы, а закрытый — строго хранится владельцем. Это позволяет любому пользователю отправить зашифрованное сообщение или подпись, но расшифровать его или проверить подпись может только обладатель закрытого ключа

<div align='center'><img src='assets/0___Визуализация_асимметричной_криптографии__модель_ключей_и_направлений_доступа.png' width='700'><br><p>Визуализация асимметричной криптографии  модель ключей и направлений доступа</p></div>

**Алгоритм:**  
1. Генерируется пара ключей: `generateKeys()` → возвращает `(public_key, private_key)`.  
2. Для шифрования данных используется открытый ключ (`encrypt(plaintext, public_key)`).  
3. Расшифровка возможна только с помощью закрытого ключа (`decrypt(ciphertext, private_key)`).  
4. Электронная подпись создаётся на основе хэша сообщения и закрытого ключа (`sign(message_hash, private_key)`), проверка — через открытый ключ (`verify(signature, message_hash, public_key)`).  

**Плюсы:**  
- Масштабируемость: не нужно хранить уникальную пару ключей для каждой пары участников.  
- Безопасность: даже при утечке открытого ключа данные остаются защищёнными.  
- Подходит для открытых систем (например, интернет), где участники заранее не известны.  

**Минусы:**  
- Медленнее симметричных аналогов из-за вычислительной сложности операций с большими числами.  
- Требует надёжного хранения закрытого ключа — его потеря означает потерю доступа к данным или подписи.  
- Уязвимость при неправильной реализации (например, использование слабых параметров или утечка приватного ключа).  

**Концепция:** RSA — один из первых и наиболее известных алгоритмов асимметричной криптографии, основанный на сложности задачи **факторизации больших чисел**. Его безопасность опирается на то, что разложение произведения двух больших простых чисел `n = p × q` на множители вычислительно невозможно за разумное время при современных технологиях

<div align='center'><img src='assets/2___График_зависимости_времени_факторизации_от_размера_числа__логарифмическая_шкала_.png' width='700'><br><p>График зависимости времени факторизации от размера числа  логарифмическая шкала </p></div>

**Алгоритм:**  
1. Генерация ключей:  
   - Выбираются два случайных простых числа `p`, `q`.  
   - Вычисляется `n = p × q`.  
   - Вычисляется функция Эйлера: `φ(n) = (p−1)(q−1)`.  
   - Выбирается число `e` (открытый показатель), взаимно простое с `φ(n)` (`gcd(e, φ(n)) == 1`).  
   - Вычисляется `d` (закрытый показатель): `d ≡ e⁻¹ mod φ(n)`.  
   - Открытый ключ: `(n, e)`, закрытый: `(n, d)`.  

2. Шифрование: `ciphertext = plaintext^e mod n`.  
3. Расшифрование: `plaintext = ciphertext^d mod n`.  

**Плюсы:**  
- Стандарт де-факто для цифровой подписи и обмена ключами.  
- Поддерживается во всех современных криптографических библиотеках (OpenSSL, Bouncy Castle и др.).  
- Простота математического обоснования — легко объяснить студентам.  

**Минусы:**  
- Уязвим к атакам при слабом выборе параметров (малые `e`, короткие `p`, `q`).  
- Не подходит для шифрования больших объёмов данных — используется только для обмена ключами или подписи коротких хэшей.  
- Подвержен квантовым атакам: алгоритм **Shor's algorithm** позволяет факторизовать числа за полиномиальное время на квантовом компьютере, что делает RSA устаревшим в постквантовой перспективе

.  

**Концепция:** Электронная подпись — механизм аутентификации и целостности сообщения, основанный на асимметричной криптографии. Подпись генерируется с использованием закрытого ключа отправителя и может быть проверена любым пользователем, имеющим его открытый ключ.

**Алгоритм:**  
1. Вычисляется хэш сообщения: `h = hash(message)`.  
2. Создаётся подпись: `signature = sign(h, private_key)`.  
3. Проверка: `is_valid = verify(signature, h, public_key)` → возвращает `true` или `false`.  

**Примеры:**  
- **RSA**: подпись — это `s = h^d mod n`.  
- **ElGamal**: используется дискретное логарифмирование в циклической группе.  
- **ECDSA (на эллиптических кривых)**: наиболее эффективный вариант для мобильных и веб-приложений.  

**Концепция:** Квантовая криптография не предназначена для шифрования данных, а служит исключительно для **безопасной передачи ключей** между сторонами. Наиболее известный протокол — **BB84**, основанный на квантовых состояниях фотонов. Любое прослушивание канала приводит к изменению состояния и обнаружению подслушивания.

**Алгоритм (BB84):**  
1. Алиса отправляет Бобу последовательность фотонов в случайных базисах (прямой или диагональный).  
2. Боб измеряет каждый фотон в случайном базисе.  
3. После передачи стороны публикуют использованные базы, но **не значения измерений**.  
4. Сохраняются только совпадения — из них формируется общий секретный ключ

.  

**Плюсы:**  
- Теоретически абсолютно безопасна против прослушивания (при условии идеальных квантовых устройств).  
- Обнаруживает любую попытку перехвата.  

**Минусы:**  
- Требует специализированного оборудования (квантовые источники, детекторы).  
- Не масштабируется на большие расстояния без повторителей.  
- Не заменяет асимметричную криптографию — используется только для генерации ключей, которые затем защищаются RSA/ECDSA.  

**Концепция:** Взлом RSA — не означает взлом самого алгоритма, а демонстрирует **практическую уязвимость реализации**, если параметры выбраны слабо. Например:  
- Малое значение `e = 3` без корректного заполнения (padding) позволяет применить **атаку по выбиванию кубического корня** (`ciphertext^(1/3) mod n`).  
- Короткие простые числа `p`, `q` — поддаются перебору или факторизации.  
- Утечка закрытого ключа через side-channel атаки (временные задержки, энергопотребление).  

**Алгоритмы взлома (упрощённо):**  
1. **Brute-force factorization**: перебор делителей для малых `n`.  
2. **Pollard's rho algorithm**: эффективен против чисел средней длины.  
3. **Wiener's attack**: если `d < n^0.25`, можно восстановить закрытый ключ по `(e, n)`.  
4. **Coppersmith method**: при наличии частичной информации о `p` или `q`.  

**Практический вывод:** Реализация собственного RSA — образовательный проект, но **не для реального использования**. Любой криптоаналитик сломает такую систему за секунды. Цель — понять, почему важно:  
- использовать проверенные библиотеки (`PyCryptodome`, `libsodium`),  
- применять padding (PKCS#1 v1.5 или OAEP),  
- выбирать параметры по стандартам (2048+ бит для `n`).  

**Концепция:** Оценка качества учебных материалов — через **рейтинги и обратную связь**, например, в формате XLS-файла. Это позволяет отслеживать:  
- количество выполненных упражнений,  
- уровень сложности,  
- частоту ошибок при реализации алгоритмов (например, взлом RSA),  
- вовлечённость студентов (дети, которые "увлекаются пару часов" — тоже метрика).

- Избегать "блеска терминов" — объяснять каждое неизвестное слово.

**Концепция:** Электронная подпись — это криптографическая конструкция, обеспечивающая аутентификацию и целостность сообщения. Она не шифрует данные, а позволяет проверить: *кто* отправил сообщение и *не было ли оно изменено*. Подпись генерируется на основе закрытого ключа отправителя и может быть проверена любым с использованием его открытого ключа

Наиболее известный протокол — **BB84** — позволяет двум сторонам сгенерировать общий секретный ключ, гарантируя, что любой прослушивающий (Eve) будет обнаружен из-за нарушения квантовых состояний

**Алгоритм:**  
1. Алиса отправляет случайные кубиты в одном из четырёх базисов (X или Z).  
2. Боб измеряет их в случайно выбранном базисе.  
3. После обмена открытыми классическими битами они отбрасывают результаты, где базис не совпал — остаётся безопасная подмножество.  
4. Из него строится общий ключ.  

**Плюсы:**  
- Теоретически **абсолютно безопасна против любого вычислителя**, включая квантовые (при условии идеальных устройств).  
- Обнаруживает прослушивание: любое измерение нарушает суперпозицию.  

**Минусы:**  
- Требует физического канала с возможностью передачи фотонов (оптика, оптоволокно).  
- Не защищает от атак на реализацию (side-channel attacks), даже если протокол BB84 корректен.  
- Не шифрует данные — только ключ для последующего симметричного шифрования (например, AES).

**Концепция:** Взлом RSA — не про "подбор" самого ключа, а про **эксплуатацию слабых параметров**, которые можно найти за секунды даже на слабом ПК. Цель — факторизовать $ N = p \cdot q $ или восстановить закрытый ключ $ d $ без знания $ p, q $.  

**Алгоритмы взлома (упражнение 8):**  
- **Wiener’s attack**: если $ d < N^{0.25} $, то можно восстановить $ d $ по $ e $ и $ N $ через непрерывные дроби.  
- **Coppersmith method**: позволяет находить небольшие корни мономов по модулю $ N $ — полезно при частичном знании закрытого ключа.  
- **Brute-force factorization** (для малых $ N $): перебор делителей до $ \sqrt{N} $.  
- **Pollard’s rho algorithm**: эффективный метод факторизации больших чисел, не зависящий от формы $ N $.

---

**Концепция:** Доказательства с нулевым знанием (Zero-Knowledge Proofs, ZKP) — это протоколы, в которых одна сторона (доказывающий) может убедить другую (проверяющего), что знает секрет (например, решение задачи), **не раскрывая сам секрет и не позволяя его воспроизвести**.  

**Пример: ZKP для знания дискретного логарифма**  
- Пусть $ g^x \equiv y \mod p $. Доказывающий знает $ x $, но не хочет его показывать.  
- Протокол:  
  1. Доказывающий выбирает случайное $ r $, отправляет $ a = g^r \mod p $.  
  2. Проверяющий просит либо:  
     - Открыть $ r $ (тогда доказывающий показывает $ r $), **или**  
     - Показать, что $ a^x = y $ (т.е. $ g^{rx} = y^r $).  
  3. Если доказывающий может выполнить запрос без знания $ x $ — он не знает логарифм.  

**Свойства ZKP:**  
- **Полнота**: если утверждение верно, честный проверяющий будет убеждён.  
- **Секретность (zero-knowledge)**: даже после бесконечного числа раундов проверяющий узнаёт только то, что утверждение истинно.  
- **Убеждённость (soundness)**: ложь не может быть принята с высокой вероятностью.  

**Применение:**  
- Аутентификация без передачи пароля.

**Концепция:** Zero-Knowledge Proof (ZKP) — это криптографический протокол, позволяющий одной стороне (доказывающей) убедить другую сторону (проверяющую), что она знает секрет (например, решение задачи), **не раскрывая сам секрет и не передавая никакой дополнительной информации**, кроме факта его существования.  
**Алгоритм:** В типичном ZKP используется тройка: *commitment* — доказательство "запечатывается" без раскрытия; *challenge* — проверяющий задаёт случайный вопрос; *response* — доказывающий отвечает так, что проверка проходит только при знании секрета. При этом повторная проверка с новым вызовом не даёт новой информации о секрете.  
**Плюсы:**

- Используется в блокчейне, децентрализованных идентификациях и безопасных голосованиях.  

**Минусы:**  
- Высокая вычислительная сложность по сравнению с классическими схемами.  
- Требует строгой реализации протокола — ошибки могут привести к утечке (например, через side-channel атаки).  
- Не все задачи допускают эффективные ZKP (ограничения по сложности доказательства).
