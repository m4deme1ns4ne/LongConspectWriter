# Conspectius Engine

## Назначение

`conspectius-engine` представляет собой программный прототип локального конвейера для автоматического построения учебного конспекта по аудиозаписи лекции. Текущая реализация предназначена для проверки работоспособности общей схемы обработки:

`аудио -> транскрипт -> черновые фрагменты конспекта -> итоговый конспект`

## Реализованный конвейер

1. Аудиофайл передаётся в модуль транскрибации.
2. Модель `faster-whisper` формирует текстовую расшифровку лекции.
3. Транскрипт разбивается на фрагменты.
4. Агент `AgentDrafter` извлекает факты и формирует набор черновых мини-конспектов.
5. Агент `AgentSynthesizer` объединяет черновики в единый итоговый конспект в формате Markdown.
6. Промежуточные и итоговые результаты сохраняются в каталоге `data/`.

## Структура проекта

- `src/core/transcribing.py` - модуль локальной транскрибации аудио.
- `src/core/agent_drafter.py` - модуль генерации черновых фрагментов конспекта.
- `src/core/agent_synthesizer.py` - модуль синтеза итогового конспекта.
- `src/ai_configs.py` - конфигурации моделей, служебные абстракции и утилиты управления памятью.
- `src/prompts/drafter.yaml` - текстовые шаблоны для LLM-агентов.
- `data/` - демонстрационные входные и выходные артефакты экспериментов.

## Тестирование

На текущем этапе автоматизированное тестирование не реализовано.

Фактически используемый способ проверки:

- запуск сквозного сценария на примерах из `data/example-audio/`;
- анализ промежуточных транскриптов из `data/example-transcrib/`;
- анализ черновых конспектов из `data/example-mini-conspect/`;
- анализ итоговых конспектов из `data/example-conspect/`.

## Ограничения текущей версии

- проект ориентирован на экспериментальный локальный запуск, а не на публикацию как завершённого программного продукта;
- интерфейс запуска пока не отделён от демонстрационных данных;
- состав моделей и параметры генерации зафиксированы в коде;
- стабильность и воспроизводимость результатов пока определяются ручной проверкой.
